Facial Expression Recognition Algorithms Test Report

1. Introduction

This report evaluates the performance of various facial expression recognition algorithms. The evaluation focuses on the accuracy and robustness. The test dataset consists of 500 randomly selected images from the FER2013 dataset. Through comparative analysis, we assess the performance of each algorithm under different scenarios.

2. Test Dataset

The FER2013 dataset is widely used in facial expression recognition. This dataset includes 35,887 grayscale images of 48x48 pixels. Each image was labeled with one of seven basic emotion categories (anger, disgust, fear, happiness, sadness, surprise, and neutral). The dataset is suitable for emotion assessment comparisons due to its diverse emotional labels and extensive image samples. We randomly selected 500 images from the FER2013 dataset as the test dataset. These images cover various emotion categories to ensure the comprehensive and representative test results.

3. Test Algorithms

Recognizing emotions from facial expressions involves two steps: face recognition and emotion detection. Our test introduced 5 face recognition algorithms (OpenCV, RetinaFace, Dlib, Yolov8 and Yunet) and 2 emotion detection algorithms (DeepFace and Facial Expression Recognition).

OpenCV is an open-source computer vision library that is widely used for image processing and computer vision tasks. It provides various algorithms for face detection and emotion analysis.

RetinaFace is a high-performance face detection algorithm capable of precise face localization and keypoint extraction. It performs well in complex scenarios but requires high-performance hardware.

Dlib is an open-source machine learning library that provides various algorithms of face detection and feature points localization. Its 68-point feature prediction model performs well in face recognition tasks.

Yolov8 is the latest version of the YOLO (You Only Look Once) series and is widely used in object detection tasks. It performs well in face detection and emotion recognition. While, this algorithm requires the support of high-performance GPU.

Yunet is a lightweight face detection algorithm for mobile and embedded systems. It performs well on low-performance devices and supports multi-platform deployment.

DeepFace is a deep learning-based library for facial recognition and emotion analysis. It supports multiple models, such as VGG-Face and Facenet, and provides high-precision results of facial feature extraction and emotion recognition.

Facial Expression Recognition (FER) is a deep learning model specifically designed for facial expression recognition. It can identify multiple emotion categories, and is well-suited for the emotion analysis tasks.

Face++ is an API of facial recognition and emotion detection. It offers the function of face detection, keypoint localization, and emotion analysis. It operates via cloud-based API calls, requiring minimal hardware resources. Face++ has the capability of both face recognition and emotion detection.

4. Accuracy Test

We conducted the accuracy test by using three indicators: emotion recognition accuracy, hardware requirement and runtime. The test results are detailed in Table 1.

4.1 Emotion Recognition Accuracy

  • Face++: 78.90% (best performance)
  • FER: 72.26%-76.84%
  • DeepFace: 64.89%-67.09%

4.2 Hardware Requirements

Ÿ   Face++: Cloud-based API calls, no strict hardware requirements, and low running costs.

Ÿ   OpenCV: Suitable for low-performance CPUs or edge devices (e.g., Raspberry Pi 4B with ARM Cortex-A72). Requires ≥ 2GB RAM, no GPU, and supports 30 FPS at 640 x 480 resolution.

Ÿ   Dlib: Suitable for low-performance CPUs or edge devices (e.g., Raspberry Pi 3B+ with a single-core CPU). Requires ≥ 1GB RAM and 500 MB of storage for the 68-point feature prediction model.

Ÿ   Yunet: Suitable for medium-performance GPU or CPU hybrid model (training: optional GPU (RTX 2060+) or CPU multi-core (i7-12700H); inference: CPU single-threaded speed 65 FPS)

Ÿ   RetinaFace: Suitable for high-performance GPU (e.g., NVIDIA RTX 3080/4090 with ≥12GB video memory for training).

Ÿ   Yolov8: Suitable for higher performance HPC GPU (training: requires RTX 3060+ GPU with ≥ 8GB of video memory (YOLOv8x version))

4.3 Runtime

The runtime for this test is the time required to process 500 images on a CPU (13th Gen Intel Core i9-13900HX) without GPU acceleration.

  • Face++: 5.18 seconds (best performance)
  • Other Algorithms: Runtime ranges from 15.88 seconds to 1233 seconds.

5. Robustness Test

We conducted the accuracy test by using two indicators: noise recognition and low-resolution emotion recognition. The test results are detailed in Table 2.

5.1 Noise Recognition

The noise recognition test evaluated each algorithm's ability to correctly identify noise in the dataset. 300 noise images (including backs, blanks, very small faces, and non-faces) were added to the test dataset.

  • RetinaFace and Yolov8: Correctly identified 298 noise samples (top performers)
  • Face++: Correctly identified 296 noise samples.

5.2 Low-Resolution Emotion Recognition

We reduced the resolution of the images in the test dataset by 50% to examine the accuracy of each algorithm's emotion recognition.

  • RetinaFace + FER: 72.73% accuracy (best performance)
  • Face++: 72.53% accuracy.

7. Conclusion

Based on the comprehensive evaluation, Face++ demonstrates good performance in accuracy, efficiency, and robustness, making it a cost-effective choice for our research. Although alternative algorithms like RetinaFace and FER show competitive performance in specific indicators, Face++ offers a balanced solution across all evaluated criteria. In addition, Face++ is better on its own than in combination with other algorithms.
